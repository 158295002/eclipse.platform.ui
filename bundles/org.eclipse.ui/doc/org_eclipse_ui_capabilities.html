Capabilities

Identifier: org.eclipse.ui.capabilities

Description: This extension point is used to register capability extensions. Capabilities are the UI equivalent of project natures, and have a one-to-one relationship with project natures. The workbench allows the user to add and remove capabilities from a project at any time.

Capabilities represent particular abilities of a project. For example, a project may have a “Java” capability (which allows the project to be compiled using a Java compiler) and/or other capabilities, depending on what is appropriate for the role of the project. If the role of the project changes over time, the project’s capabilities can be modified to suit the change. For example, a project may start as a simple Java project, but may later gain a new requirement to use JNI to some legacy library. The user may then add a “C Development” capability (if one exists) to the project so it can meet this new requirement.

Some capabilities may wish to handle the user interface for other capabilities. This generally occurs when one capability’s corresponding nature is dependant on the other capability’s corresponding nature.

The categories defined by one plug-in can be referenced by other plug-ins using the category attribute.

Configuration Markup:

   <!ELEMENT category EMPTY>
   <!ATTLIST category
      id             CDATA #REQUIRED
      name           CDATA #REQUIRED
   >

  • id - a unique name that can be used to identify this category.
  • name - a translatable name of the category.
   <!ELEMENT capability (handleUI)?>
   <!ATTLIST capability
      id                 CDATA #REQUIRED
      natureId           CDATA #REQUIRED
      category           CDATA #OPTIONAL
      icon               CDATA #OPTIONAL
      installWizard      CDATA #REQUIRED
      installDetails     CDATA #OPTIONAL
      description        CDATA #OPTIONAL
   >
  • id - a unique name that can be used to identify this wizard.
  • natureId - a unique identifier of the project nature associated with this project capability.
  • category - an optional attribute which defines a slash-delimited path ('/') of category IDs. Each token in the path must represent a valid category ID previously defined by this or some other plug-in. If omitted, the capability will be added to the "Other" category.
  • icon - an optional attribute which defines a relative path of the icon that may be used to represent the capability.
  • installWizard - a fully qualified name of the Java class implementing org.eclipse.ui.ICapabilityWizard.
  • installDetails - an optional attribute which decribes briefly to the user what will eb required of them by the install wizard for the capability.
  • description - an optional attribute which consists of a short description of what functionality or ability the capability adds to a project.
  • handleUI - an optional subelement whose body contains a unique identifier for another capability whose UI will be controlled by the UI of this capability.
   <!ELEMENT handleUI EMPTY>
   <!ATTLIST handleUI
      id  CDATA #REQUIRED
   >
  • id - the unique identifier of another capability for which the capability which is the parent of this element controls the UI.
Examples:

Following is an example of capability configuration:

   <extension point="org.eclipse.ui.capabilities">
      <category
        id="com.xyz.weather"
        name="Weather Elements">
      </category>
      <capability
          id="com.xyz.snowCapability"
          natureId="com.xyz.snowNature"
          category="com.xyz.weather"
          icon="./icons/snowCapability1.gif"
          installWizard="com.xyz.snowCapabilityWizard">
          installDetails="You will be asked to supply a locale id.">
          description="Turn your project into a winter wonderland!">
          <handleUI
          id="com.xyz.waterCapability"
          </handleUI>
      </capability>
      <capability
          id="com.xyz.waterCapability"
          natureId="com.xyz.waterNature"
          category="com.xyz.weather"
          icon="./icons/waterCapability1.gif"
          installWizard="com.xyz.waterCapabilityWizard">
          installDetails="You will be asked to supply a locale id.">
          description="Turn your project into a watery wonderland!">
      </capability>
   </extension>

API Information: The value of the installWizard attribute must represent a class that implements org.eclipse.ui.ICapabilityWizard.

Supplied Implementation: The workbench comes with Java and PDE capabilities, and install wizards for these capabilities.
